# 蓝区开源社区开发全流程指导文档

## 文档概述

本文档旨在为蓝区开源社区的开发者提供完整的开发流程指导，涵盖从环境准备到项目发布的全生命周期管理。通过建立标准化的工程能力要求和开发流程，确保开源项目的高质量、高效率交付。

---

## 一、工程能力要求

本章节定义了蓝区开源社区项目的工程能力标准，所有项目应达到以下要求：

| 序号 | 能力要求 | 详细说明 |
|------|----------|----------|
| 1.1 | 编码风格可视化 | .clang-format/.clang-tidy/.ruff.toml |
| 1.2 | 代码检查（pre-commit） | pre-commit-config.yaml，pr流水线被本地规则100%覆盖 |
| 1.3 | 本地增量编译<30s | 增量编译时间控制在30秒以内 |
| 1.4 | UT执行时长，单个UT<30s | 单个单元测试执行时间不超过30秒 |
| 1.5 | 代码检查规则数对齐业界标杆社区 | 代码检查规则数量和质量与业界标杆开源社区保持一致 |
| 1.6 | 代码风格自动修复 | pr流水线提供自动化的代码风格修复能力 |
| 1.7 | AI辅助检视 | gitcode codereview |
| 1.8 | 例外代码检查规则备案和committer审核一体化 | 建立例外规则的备案流程，以代码化注释的方式屏蔽代码检查规则，如有规则屏蔽，则机器人自动提示committer |
| 1.9 | PR执行时长 | 通算场景<20分钟，智算场景<1小时 |
| 1.10 | PR资源消耗 | 对接openLibing看板，实时显示PR资源消耗情况，提升资源透明度 |
| 1.11 | 编译成功率100% | 保证主干代码的编译成功率达到100% |
| 1.12 | 版本可用度>90% | 确保发布版本的可用度超过90% |
